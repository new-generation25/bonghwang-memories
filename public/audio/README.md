# 내레이션 음성 파일

이 폴더에 음성 파일을 넣으면 앱이 자동으로 인식해 재생 컨트롤을 표시합니다.
**파일이 없으면 컨트롤 자체가 나타나지 않으므로**, 준비된 것부터 하나씩 넣으셔도 됩니다.

## 파일명 규칙

파일명이 코드의 ID와 정확히 일치해야 합니다. 대소문자와 하이픈까지 같아야 합니다.

| 파일명 | 화자 | 내용 | 대본 위치 |
|---|---|---|---|
| `prologue.m4a` | 아빠 | 프롤로그 편지 (378자 · 약 84초) | `src/app/story/page.tsx`의 `storyText` |
| `mission-main-1-outro.m4a` | 소영 | 1번 미션 완료 | `src/lib/missions.ts` → `main-1`의 `story.outro` |
| `mission-main-2-outro.m4a` | 소영 | 2번 미션 완료 | 〃 `main-2` |
| `mission-main-3-outro.m4a` | 소영 | 3번 미션 완료 | 〃 `main-3` |
| `mission-main-4-outro.m4a` | 소영 | 4번 미션 완료 | 〃 `main-4` |
| `mission-main-5-outro.m4a` | 소영 | 5번 미션 완료 | 〃 `main-5` |
| `mission-sub-1-outro.m4a` | 소영 | 보너스 미션 완료 | 〃 `sub-1` |
| `mission-sub-2-outro.m4a` | 소영 | 보너스 미션 완료 | 〃 `sub-2` |

`.m4a`가 없으면 같은 이름의 `.mp3`를 찾습니다. 둘 다 없으면 조용히 건너뜁니다.

미션 진입 시 안내(`mission-{id}-intro`)와 완주 축하(`complete`)도 코드가 지원하지만
아직 화면에 연결하지 않았습니다. 필요해지면 연결하면 됩니다.

## 권장 사양

- **형식**: AAC(.m4a) 또는 MP3
- **비트레이트**: 64kbps 모노 — 음성에는 충분하고 용량이 작습니다
- **샘플레이트**: 44.1kHz
- **볼륨**: -16 LUFS 정도로 통일. 클립마다 크기가 다르면 야외에서 듣기 나쁩니다
- **앞뒤 무음**: 0.3초 이내로 잘라주세요. 길면 반응이 굼떠 보입니다

전체 분량이 4분 남짓이라 모두 합쳐도 **약 2MB**입니다. Vercel이 정적 파일로
서빙하므로 Firebase Storage(Blaze 요금제)와 무관하게 동작합니다.

## 제작 방법

**아빠 목소리 (prologue)** — 성우 녹음을 권합니다. 딸에게 남기는 마지막 편지라
서비스의 감정적 핵심이고, 84초 분량이라 비용 부담도 크지 않습니다.

**소영 목소리 (미션 안내)** — TTS로 생성해도 충분합니다. 기능적 안내이고
대사가 바뀌어도 재생성이 쉽습니다. 한국어 TTS 선택지:

- **네이버 클로바 보이스** — 이미 네이버 클라우드 계정을 쓰고 있어 추가 가입이 없습니다
- **구글 Cloud TTS** — Firebase와 같은 GCP 프로젝트에서 바로 사용 가능
- **일레븐랩스 · 타입캐스트** — 감정 표현이 더 자연스럽습니다

## 카세트 질감 (선택)

브랜드 콘셉트가 '1988년에 녹음된 테이프'이므로, 아빠 목소리에 약한 테이프
잡음과 살짝의 고역 감쇄를 입히면 몰입이 크게 올라갑니다. 소영 목소리는
현재 시점의 안내이므로 깨끗하게 두어 대비를 만드는 편이 좋습니다.

## 넣은 뒤 확인

파일을 넣고 개발 서버를 재시작한 뒤 `/story`에 들어가면 편지 위에
재생 컨트롤이 나타납니다. 나타나지 않으면 파일명 오타일 가능성이 높습니다.
